# Dataset — full-load (devices reference)

Consumed by: **Day 01 — Full Loader**.

## Source
Synthetic. Generated by `generate.py` (deterministic, seed = 42). No real PII.

## Why this shape
The Full Loader pattern applies to slowly-changing reference data that has **no delta column**.
So the dataset is a `devices` reference table delivered as **full snapshots** — there is no
`updated_at`, which is exactly the condition that forces a full reload.

## Files
| File | Rows | Meaning |
|------|------|---------|
| `devices_snapshot_1.csv` | 500 | the reference dataset on day 1 |
| `devices_snapshot_2.csv` | 520 | a later delivery: 30 rows silently changed in place + 20 new devices |

Because there is no column telling us which 30 rows changed, the only correct ingestion is to
reload all 520 rows.

## Schema
| Column | Type | Example |
|--------|------|---------|
| `device_id` | string | `DV00042` |
