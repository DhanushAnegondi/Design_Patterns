# Dataset — incremental-load (visit events)

Consumed by: **Day 02 — Incremental Loader**.

## Source
Synthetic. Generated by `generate.py` (deterministic, seed = 7). No real PII.

## Why this shape
The Incremental Loader needs an append-only, growing source with a usable **delta column**. So the
data is immutable `visit` events carrying an `event_time`. The batches simulate the source growing
between runs, and a deliberate straggler demonstrates the late-data hazard.

## Files
| File | Rows | Meaning |
|------|------|---------|
| `visits_batch_1.csv` | 300 | first wave, event_time within hour 09 |
| `visits_batch_2.csv` | 200 | later arrivals, event_time within hour 10 |
| `visits_late.csv` | 1 | a straggler whose event_time falls back in hour 09, delivered last |

## Schema
| Column | Type | Example |
|--------|------|---------|
| `visit_id` | string | `V000042` |
| `event_time` | ISO-8601 string | `2026-06-12T09:30:29` |
| `user_id` | string | `U0087` |
| `page` | string | `/post/idempotency` |

`event_time` is ISO-8601 so it sorts/compares correctly as a plain string (the loader filters
`event_time > watermark` lexically).
