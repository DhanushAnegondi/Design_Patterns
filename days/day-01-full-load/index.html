The Full Loader, and the art of the atomic swap

When reference data has no "what changed" column, you reload all of it every run. Computing the reload is easy. Publishing it without ever showing a consumer a half-written dataset is the real skill, and on S3 it comes down to one pointer write.

pattern: Full Loader store: Amazon S3 local: moto mock lang: Python · boto3

The shape of the problem

Three blocks, the way the book frames every pattern.

Problem

A device reference dataset from an external provider feeds the Silver layer. It changes a few times a week, stays under a million rows, and ships with no "last updated" column. You can't tell which rows changed, so you can't load incrementally.

Solution

Reload the entire dataset each run (extract + load). Publish it atomically: write the new snapshot to an immutable version, then flip a single _CURRENT pointer. Readers always resolve a complete version.

Consequences

You reprocess everything every run: fine for small, slow data, costly for big data. Never drop-then-insert in place, because it opens a window where consumers see zero rows. Keep old versions for rollback.

How the safe load flows on S3

The new snapshot lands in an immutable versions/<v>/ prefix that consumers aren't watching. Only when it's fully written does the pointer flip. That flip is the object-store version of swapping a database view.

The mechanism, in code

The whole safety property is two ordered writes. From code/full_loader.py.

# write the full snapshot to an immutable location nobody is reading yet
self.s3.put_object(Bucket=bucket, Key=f"{prefix}/versions/{version}/data.csv", Body=csv_bytes)

# then publish it with ONE atomic pointer write — this is the "view swap"
self.s3.put_object(Bucket=bucket, Key=f"{prefix}/_CURRENT", Body=version.encode())

# a reader always resolves _CURRENT first, so it only ever sees a COMPLETE version
def read_current_count(self):
    version = self.current_version()
    obj = self.s3.get_object(Bucket=bucket, Key=f"{prefix}/versions/{version}/data.csv")
    return count_rows(obj["Body"].read())
